Architecture

4-module clean architecture

Concerns are razor-separated. Every screen follows a strict MVI contract — predictable, testable, and scales cleanly.

:app Entry point. Navigation host, Hilt setup, Activity, and Application class.
:data Room DB, Firebase repositories, DataStore prefs, WorkManager workers, and all data models.
:presentation All screens, ViewModels, MVI State/Event/Effect contracts, and Compose UI trees.
:designsystem Shared theme, typography, colours, reusable Compose components, and Canvas chart primitives.

⚡ MVI Contract per Screen

S

State — Single source of truth for the entire screen. Immutable data class emitted as a Flow.

↓
E

Event — Sealed class of all possible user interactions the screen can receive from the UI.

↓
EF

Effect — One-time side effects like navigation, showing a snackbar, or triggering a dialog.
